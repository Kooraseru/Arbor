# Arbor の例

この `content/jp/examples` は、Arbor のドキュメントとアナライザーチェックで使う小さなソースフィクスチャです。

それぞれの例では、ひとつのことだけがはっきり分かるようにします：

```txt
問題
ソース
期待される型の挙動
必要な場合はランタイム境界
```

## 例

- `direct-children/` は、直接の子の名前、直接の子のルックアップ、子名をキーにしたレコードを確認します。
- `class-filtered-children/` は、クラスで絞り込む直接の子ヘルパーを確認します。
- `runtime-loader/` は、明示的な検証つきの動的な `ModuleScript` ロードを確認します。

フィクスチャの取り決めと必要なアナライザーチェックは、`../wiki/reference/analyzer-fixtures.md` を見てください。

## スクリーンショットとチュートリアル用メモ

スクリーンショットやチュートリアル動画は、一回きりのスニペットではなく、チェック済みの例ファイルから撮ってください。

まわりのドキュメントがある程度固まって、すぐ古くならない状態になってから撮るのが安全です。
