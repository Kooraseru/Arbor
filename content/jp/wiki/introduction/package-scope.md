# パッケージスコープ

Roblox のパッケージコードには、よく 2 つの競合する要求があります：

```txt
モジュールを子として整理したい
手で管理する文字列マニフェストは避けたい
```

Arbor は、その問題のうちアナライザーから見える部分を扱います。Roblox のインスタンス階層を Luau の型システムに公開し、その形を土台にして、パッケージコードがコンパイル時の基盤を作れるようにします。

## 何に役立つか

- 直接の子の名前を取り出す
- 正確な子キーを持つレコードを作る
- アナライザーから見えるクラスで子を絞り込む
- ランタイムロードと検証を組み合わせる
- アナライザーが実際に見ているツリーを通して、パッケージ境界をドキュメント化する

## 避けること

Arbor は、あなたのレジストリ、ライフサイクル、パッケージマネージャー、ディスパッチ層、シリアライザーにはなりません。

この境界は大事です。Arbor はパッケージの静的ツリーを記述する助けになるべきであり、そのパッケージのランタイム挙動まで所有するべきではありません。
